# 智能家庭菜单规划与采买清单 (Smart Menu Planner)

这是一个极轻量、高度定制化的家庭菜谱排班工具。采用纯 HTML+CSS(Tailwind)+JS(Vanilla) 开发并支持部署到 GitHub Pages。

它的主要使用场景是帮助雇主轻松管理由“自家人+外籍女佣”构成的多元家庭饮食结构。雇主只需日常维护一个 Google Sheets 表格，网页即能一键生成合理规划的、带有中缅（多国）双语对照的菜单和自动合并同类项的采购清单（Shopping List）。

## 核心能力

- **多模式菜单排班**：自动生成午餐（默认2-3人不含汤）与晚餐（默认全家带汤）并进行随机组合，同时智能监控“每日重口味菜数量”，有效平衡重油重辣与清淡食物的分布。
- **免数据库管理菜单**：直接抓取本地或线上发布的同名 `sample-data.csv`，可以随时靠修改表格扩展您的微型食谱库。
- **智能食材折算与聚合系统**：内置基于参数的人数折算器，将多顿菜单所需消耗的同一食材，按其人数占比智能放缩后累加计算（午餐和晚餐用到的同一个料可以汇总），最终输出完美双语 Checklist 交由女佣按表购物。

## 如何使用 LLM 帮忙快速录入新菜品 (Prompt Rule)

您可以将下面这段“**提示词（Prompt）**”直接发送给 ChatGPT / Claude 等大语言模型，并告诉它你想生成什么菜。模型按照下面这些限制给出结果后，您就可以直接粘入您的 Google Sheet / CSV 中，让您的个人菜单库快速变强！

### 【复制以下文本发送给 LLM】

> 你现在是我的家庭食谱数据录入助手。我会给你报一道或者几道菜名，你需要按照下方的严格规则，帮我输出完美对应数量的 CSV 格式菜谱数据，绝不能有偏差。
>
> **【数据列的顺序】**
> `id, name_zh, name_my, ingredients_zh, ingredients_my, category, difficulty, heaviness, base_portions, image_url, steps_zh, steps_my`
>
> **【各数据列格式的苛刻规则，严格执行】**
> 1. `id`: 帮我按顺序编排一个唯一编号。
> 2. `name_zh`: 菜名的正确中文（如：蒜蓉西兰花）。
> 3. `name_my`: 菜名地道的**缅甸文翻译**。
> 4. `ingredients_zh`: 中文食材及数量列表。**极其核心规则**：必须严格遵循`食材名:数字:纯英文单位` 的三段式格式，如果一道菜有多个食材，用**英文分号 `;`** 隔开！
>    - 例如正确写法：`西蓝花:1:pc;大蒜:3:clove;食盐:5:g;耗油:10:ml`。
>    - 单位必须全是英文短写（`pc`, `box`, `clove`, `g`, `ml`, `L`, `pack`, `bottle`, `bunch`）。
> 5. `ingredients_my`: 对应食材名翻译成**缅甸文**。结构必须和中文绝对对应，数量和英文单位原封不动抄过来！
>    - 例如：`ပန်းဂေါ်ဖီစိမ်း:1:pc;ကြက်သွန်ဖြူ:3:clove;ဆား:5:g;ခရုဆီ:10:ml`
> 6. `category`: 只能是：`荤菜`，`素菜`，`半荤半素`，`汤`，`主食` 的其中之一。
> 7. `difficulty`: 烹饪对家政工的难度，填写 `1` 到 `5` 的整数。
> 8. `heaviness`: 菜品的油腻口味程度，只能是：`清淡`，`适中`，`重口` 的其中之一。
> 9. `base_portions`: 是上方填写的食材量能供几个人吃，默认统一写 `5`即可。
> 10. `image_url`: 菜品封面图链接。如果您能找到公开的图就填入，否则**必须留空**。
> 11. `steps_zh`: 制作这道菜的中文核心步骤。用英文分号`;`隔开。（例如：`1. 豆腐切块待用; 2. 锅中烧油爆香肉末; 3. 加入豆腐慢炖出锅`）
> 12. `steps_my`: 制作这道菜的地道缅甸语步骤。条数和排序必须和中文一对一，同样用英文分号`;`隔开。
>
> 明白了吗？如果明白了，请直接在下一轮对话准备接单，每轮给出 CSV 原生文本（不带 markdown tag）。

## 部署与本地开发指南

由于本网页无任何环境依赖，极其省事：

**1. 本地浏览**
直接用浏览器双击打开项目中的 `index.html` 即可使用。亦可使用 Python 运行一个临时服：
```bash
python3 -m http.server 8000
```
然后在浏览器访问 `http://localhost:8000`。

**2. 借助 Github Pages 上线 (供女佣手机访问)**
```bash
git add .
git commit -m "update intelligent planner"
git push origin main
```
在 Github 的代码仓库 -> **Settings** -> **Pages** 将 `Branch` 设为 `main` 并保存，喝口水后就能用生成的专属网址在手机愉快查表啦。
